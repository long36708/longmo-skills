# 优先选择可点击节点

## 问题描述

当选择器匹配到多个节点时，如果直接点击文本节点而不是可点击的容器节点，可能会导致点击失败或无法触发预期行为。

## 错误示例

```text
[text="我不喜欢这个广告"]
```

## 正确示例

```text
@[clickable=true] > [text="我不喜欢这个广告"]
```

## 说明

- **优先选择 `clickable=true` 的节点**：如果节点符合 `clickable=true`，则向系统发起一个点击无障碍节点事件。即使节点在屏幕外部或者被其它节点遮挡，也依然能够正确触发点击目标节点。
- **避免直接点击文本节点**：文本节点通常不是可点击的容器，直接点击可能会失败。
- **使用 `@` 标记目标节点**：明确标记需要点击的可点击容器节点。

## 实际应用

对于快照 https://i.gkd.li/i/12798810，应该选择可点击的容器节点而不是文本节点。
