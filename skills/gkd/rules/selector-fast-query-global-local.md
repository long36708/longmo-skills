# 全局与局部快速查询

## 问题描述

选择器查询性能差异很大，不理解快速查询机制会导致规则查询缓慢或查询失败。

## 错误示例

```text
A > B + C[childCount=2][id='x'] <<n D
```

## 正确示例

### 全局快速查询

```text
A > B + C[id='x'][childCount=2]
```

### 局部快速查询

```text
A > C[id='x'] <<n D[id='y'] <<n E
```

## 说明

### 全局快速查询条件

- **选择器从右往左查询**：快速查询需要满足选择器末尾属性选择器的第一个属性表达式符合特定格式
- **快速查询属性**：`[id='abc']`, `[vid='abc']`, `[text='abc']`, `[text^='abc']`, `[text*='abc']`, `[text$='abc']`
- **逻辑表达式支持**：支持使用 `||` 连接多个快速查询条件

### 局部快速查询条件

- **使用 `<<n` 操作符**：左侧的选择器必须满足快速查询格式
- **多个局部查询**：可以链式使用多个局部快速查询

### 启用快速查询

需要在规则或规则组中设置 `fastQuery: true` 才会启用快速查询。

### 避免快速查询误用

如果节点无法快速查询，即使选择器符合快速查询格式，也不要设置 `fastQuery: true`，会查询不到。

## 实际应用

使用快照工具的快速复制规则按钮，可以快速获取可快速查询的节点到当前节点的表达式。
