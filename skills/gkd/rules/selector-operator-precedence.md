# 理解选择器操作符优先级

## 问题

在复杂的逻辑表达式中，不了解操作符优先级可能导致意外的匹配结果。

## 错误示例

```text
// BUG: 优先级问题，实际等价于 a>1 || (b>1 && c>1) || d>1
[a>1 || b>1 && c>1 || d>1]

// BUG: 并列的 [] 被视为 && 连接
[a=1][b=1]  // 等价于 [a=1&&b=1]
```

## 正确示例

```text
// 明确分组，确保逻辑正确
[a>1 || (b>1 && c>1) || d>1]

// 或者使用并列 [] 表示 &&
[a=1 && b=1]  // 明确使用 &&

// 复杂的逻辑组合
[(a=1 || a=2) && (b=3 || b=4)]
```

## 操作符优先级规则

### 从高到低的优先级

1. **并列属性选择器** (`[a][b]` = `[a&&b]`)
2. **逻辑与** (`&&`)
3. **逻辑或** (`||`)
4. **关系操作符** (`+`, `-`, `>`, `<`, `<<`)

### 逻辑表达式优先级

```text
// 高优先级：&&
[a=1 && b=2 || c=3]  // 等价于 [(a=1 && b=2) || c=3]

// 低优先级：||
[a=1 || b=2 && c=3]  // 等价于 [a=1 || (b=2 && c=3)]
```

## 实际应用场景

### 条件组合

```text
// 匹配特定文本的按钮，且可点击
@TextView[(text='确认' || text='确定') && clickable=true]

// 匹配特定 ID 或特定文本的节点
[vid='close_btn' || (text*='关闭' && clickable=true)]
```

### 复杂条件

```text
// 匹配广告相关的节点
@LinearLayout[(desc*='广告' || text*='广告') && visibleToUser=true]

// 匹配特定范围内的文本长度
TextView[text.length>2 && text.length<10 && text*='跳过']
```

## 调试技巧

1. **使用括号明确分组**: 不确定优先级时使用括号
2. **逐步测试**: 先测试简单表达式，再组合复杂逻辑
3. **快照工具验证**: 在网页审查工具中验证匹配结果

## 常见错误模式

### 错误：误用并列选择器

```text
// BUG: 这表示必须同时满足两个条件
[a=1][b=1]  // 等价于 [a=1 && b=1]

// 正确：如果需要或关系
[a=1 || b=1]
```

### 错误：混合使用不同优先级

```text
// BUG: 可能导致意外结果
[a=1 || b=1 && c=1]

// 正确：明确分组
[a=1 || (b=1 && c=1)]
```

## 原因

理解操作符优先级可以：
1. **避免逻辑错误**: 确保选择器按预期工作
2. **提高可读性**: 明确的逻辑结构便于维护
3. **减少调试时间**: 快速定位逻辑问题
