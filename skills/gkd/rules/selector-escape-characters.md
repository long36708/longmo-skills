# 正确处理嵌套转义字符

## 问题

在多层环境中（GKD 选择器 → JSON 规则 → 正则表达式），转义字符需要正确嵌套处理，否则会导致语法错误或匹配失败。

## 错误示例

```text
// BUG: 直接在 GKD 规则中输入，缺少 JSON 层转义
TextView[text="\n"]

// BUG: 正则表达式缺少转义
TextView[text~="\d"]
```

## 正确示例

```text
// 匹配包含换行符的文本
TextView[text="\\n"]

// 匹配数字的正则表达式
TextView[text~="\\d"]

// 匹配反斜杠字符
TextView[text~="\\\\"]
```

## 转义层级说明

### 第一层：GKD 选择器

```text
// 在 GKD 选择器中直接使用
TextView[text="\\n"]  // 实际匹配文本：\n
TextView[text~="\\d"] // 实际匹配正则：\d
```

### 第二层：JSON 规则（需要额外转义）

```json5
{
  // JSON 层需要再次转义反斜杠
  matches: 'TextView[text="\\\\n"]',        // 最终匹配：\n
  matches: 'TextView[text~="\\\\d"]',       // 最终匹配：\d
  matches: 'TextView[text~="\\\\\\\\"]', // 最终匹配：\\
}
```

## 常见转义需求

### 匹配特殊字符

| 字符 | GKD 选择器 | JSON 规则 |
|------|------------|-----------|
| 换行符 | `"\\n"` | `'"\\\\n"'` |
| 制表符 | `"\\t"` | `'"\\\\t"'` |
| 反斜杠 | `"\\\\"` | `'"\\\\\\\\"'` |
| 双引号 | `"\\""` | `'"\\\\\""'` |

### 正则表达式转义

| 用途 | GKD 选择器 | JSON 规则 |
|------|------------|-----------|
| 数字 | `"\\d"` | `'"\\\\d"'` |
| 空白 | `"\\s"` | `'"\\\\s"'` |
| 单词 | `"\\w"` | `'"\\\\w"'` |

## 调试技巧

1. **使用快照工具**: 在网页审查工具中测试选择器
2. **逐层验证**: 先在 GKD 层测试，再应用到 JSON 规则
3. **查看实际值**: 使用 `text.length` 等属性验证实际匹配内容

## 原因

正确的转义处理确保：
1. **语法正确性**: 避免 JSON 解析错误
2. **匹配准确性**: 确保正则表达式按预期工作
3. **跨平台一致性**: 在 GKD 和网页工具中表现一致
